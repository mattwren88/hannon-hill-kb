curl -X POST "https://your-instance.cascadecms.com/api/v1/read" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "authentication": {
      "apiKey": "your-api-key"
    },
    "identifier": {
      "type": "page",
      "path": { "path": "/index", "siteName": "my-site" }
    }
  }'
import requests

url = "https://your-instance.cascadecms.com/api/v1/read"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
payload = {
    "authentication": {"apiKey": "your-api-key"},
    "identifier": {
        "type": "page",
        "path": {"path": "/index", "siteName": "my-site"}
    }
}

response = requests.post(url, json=payload, headers=headers)
data = response.json()
print(data["asset"]["page"]["title"])
import com.hannonhill.cascade.api.*;

CascadeAPI api = new CascadeAPI("https://your-instance.cascadecms.com");
api.authenticate("your-api-key");

ReadResult result = api.read(
    new Identifier("page", "/index", "my-site")
);

Page page = result.getAsset().getPage();
System.out.println(page.getTitle());

Numbered Steps

1

Get your API key

Navigate to your user profile in Cascade CMS and generate an API key under the API Keys tab. Copy the key and store it securely.

2

Set up your environment

Store your API key as an environment variable for secure access:

Shell 
export CASCADE_API_KEY='your-api-key-here'
3

Make your first API call

Test the connection by reading a page from your site. The API returns the full asset object including metadata, content, and system properties.

4

Handle the response

Parse the JSON response and extract the data you need. Check the success field to verify the operation completed successfully before processing the asset data.

Content Tabs

On Windows, download the MSI installer from the Cascade CMS downloads page. Run the installer with administrator privileges and follow the setup wizard. The default installation path is C:\Program Files\Cascade CMS.

After installation, the Cascade CMS service will start automatically. Access the admin console at https://localhost:8443.

On macOS, download the DMG package from the Cascade CMS downloads page. Mount the disk image and drag the application to your /Applications folder.

Open Terminal and run sudo /Applications/CascadeCMS/bin/start.sh to start the server. Access the admin console at https://localhost:8443.

On Linux, download the tar.gz archive and extract it to your preferred directory (commonly /opt/cascade). Ensure Java 11+ is installed and available on your PATH.

Run ./bin/start.sh from the installation directory. Configure systemd for automatic startup. Access the admin console at https://localhost:8443.
